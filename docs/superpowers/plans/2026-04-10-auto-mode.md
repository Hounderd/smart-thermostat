# Auto Mode Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a single-target `AUTO` thermostat mode that heats, cools, or idles automatically based on the current temperature and existing deadband.

**Architecture:** Extend the thermostat engine so `AUTO` resolves into a concrete active call (`HEAT` or `COOL`) while preserving existing compressor lockout, history logging, and optimistic dashboard control flow. Keep the API payload shape unchanged and add the new mode to the dashboard without introducing dual setpoints.

**Tech Stack:** Python, FastAPI, React, plain `unittest`, Node-based dashboard helper tests
